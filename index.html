Simple Present Tense

Used for habits, facts, and regular actions

Structure: Subject + Verb (s/es) + Object
(+) "She reads books every day."
(-) "She does not read books."
(?) "Does she read books?"

Simple Past Tense

Used for completed actions in the past

Structure: Subject + Verb2 (past form) + Object
(+) "I visited Jakarta last week."
(-) "I did not visit Jakarta."
(?) "Did you visit Jakarta?"

Present Continuous Tense

Used for actions happening right now

Structure: Subject + is/am/are + Verb-ing
(+) "They are studying English now."
(-) "They are not studying."
(?) "Are they studying?"

Present Perfect Tense

Used for past actions with present relevance

Structure: Subject + have/has + Verb3 (past participle)
(+) "I have finished my homework."
(-) "I have not finished yet."
(?) "Have you finished?"

Future Tense (Will)

Used for predictions, promises, and spontaneous decisions

Structure: Subject + will + Verb (base form)
(+) "I will help you tomorrow."
(-) "She will not come to the party."
(?) "Will they join us?"

Passive Voice

Used when the action is more important than who did it

Structure: Object + to be + Verb3 (+ by subject)
Active: "The chef cooks the food."
Passive: "The food is cooked by the chef."
Past: "The book was written in 2020."

Conditional Sentences

Used for hypothetical situations

Type 1 (Real): "If it rains, I will stay home."
Type 2 (Unreal): "If I were rich, I would travel."
Type 3 (Past): "If I had studied, I would have passed."

Functions & Arrow Functions

Functions are reusable blocks of code. Arrow functions are shorter syntax.

JavaScript
// Regular function
function calculateArea(width, height) {
    return width * height;
}

// Arrow function
const multiply = (a, b) => a * b;

// Function with default parameters
const greet = (name = "World") => `Hello, ${name}!`;

// Higher-order function
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(n => n * 2); // [2, 4, 6, 8, 10]
const evens = numbers.filter(n => n % 2 === 0); // [2, 4]
const sum = numbers.reduce((acc, n) => acc + n, 0); // 15

Fetch API & Async/Await

Modern way to make HTTP requests and handle asynchronous operations.

JavaScript
// Fetch API with async/await
async function getUsers() {
    try {
        const response = await fetch('https://api.example.com/users');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Error:', error);
    }
}

// POST request
async function createUser(userData) {
    const response = await fetch('/api/users', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(userData)
    });
    return await response.json();
}

Control Flow & Loops

Control the flow of your program with conditions and loops.

Python

Functions & Classes (OOP)

Organize your code with functions and object-oriented programming.

Python
# Functions with default params
def calculate_bmi(weight, height_m):
    """Calculate Body Mass Index"""
    return round(weight / (height_m ** 2), 1)

print(calculate_bmi(70, 1.75))  # 22.9

# Class (OOP)
class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade
        self.courses = []

    def enroll(self, course):
        self.courses.append(course)
        return f"{self.name} enrolled in {course}"

    def __str__(self):
        return f"Student({self.name}, Grade {self.grade})"

# Usage
student = Student("Alice", 10)
print(student.enroll("Math"))
print(student)

CRITICAL

SQL Injection (SQLi)

Injecting SQL code into queries to access/modify database.

Example Payload
' OR 1=1 --
' UNION SELECT username,password FROM users --
admin' AND 1=1 --

Impact: Data breach, authentication bypass, full database access

HIGH

Cross-Site Scripting (XSS)

Injecting malicious scripts that execute in victim's browser.

Types of XSS
Reflected: <script>alert('XSS')</script>
Stored: Saved in database, executes for all users
DOM-based: Manipulates client-side JavaScript

Impact: Session hijacking, cookie theft, phishing

HIGH

Server-Side Request Forgery (SSRF)

Making the server send requests to internal resources.

Example
url=http://169.254.169.254/latest/meta-data/
url=http://localhost:8080/admin
url=http://internal-service.local/api/secrets

Impact: Internal network access, cloud metadata exposure, RCE

MEDIUM

Insecure Direct Object Reference (IDOR)

Accessing other users' data by changing ID parameters.

Example
GET /api/user/profile?id=123  (your profile)
GET /api/user/profile?id=124  (someone else's!)
GET /api/invoice/download?id=5001 (other user's invoice)

Impact: Unauthorized data access, privacy violation

CRITICAL

Remote Code Execution (RCE)

Executing arbitrary commands on the target server.

Example
; cat /etc/passwd
| whoami
`id`
${7*7} (Template injection -> RCE)

Impact: Full server compromise, data exfiltration

VLOOKUP & HLOOKUP

Look up values in tables - the most asked formula in interviews!

Formula
=VLOOKUP(lookup_value, table_array, col_index, [exact_match])

Example: Find employee salary by ID
=VLOOKUP(E2, A2:C100, 3, FALSE)

- E2: The ID to search for
- A2:C100: The data table (ID must be in first column)
- 3: Return value from 3rd column (Salary)
- FALSE: Exact match only

Alternative (Modern): INDEX + MATCH
=INDEX(C2:C100, MATCH(E2, A2:A100, 0))
This is more flexible - lookup column doesn't need to be first!

Pivot Tables

Summarize large datasets in seconds. The most powerful Excel feature.

Steps to Create Pivot Table
1. Select your data (Ctrl + A)
2. Insert > PivotTable
3. Choose where to place it (New Worksheet recommended)

Calculate areas of common shapes

Pythagorean Theorem

For right triangles: a² + b² = c²

a² + b² = c²

Given: a = 3, b = 4

Step 1: 9 + 16 = c²

Step 2: c² = 25

Answer: c = 5

Calculate volume of 3D shapes

Central Tendency

Mean, Median, and Mode

Data: 3, 5, 7, 7, 9, 11

Mean: (3+5+7+7+9+11)/6 = 7

Median: (7+7)/2 = 7

Mode: 7 (most frequent)

Range: 11-3 = 8

Coin heads: P = 1/2

Card (ace): P = 4/52 = 1/13
